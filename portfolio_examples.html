►OpenGL Particle System - GitHub

Classwork | January - April 2024

This project was developed as part of my advanced graphics course during my final semester at UPEI.

I implemented a particle system using linked list memory management to create a dynamic pool of particles, enabling efficient addition and removal as needed. The system was designed to be data-driven, allowing for easy expansion through XML without requiring additional code modifications.

To demonstrate the system's versatility, I created two distinct visual effects: one featuring two emitters for smoke and fire, and another using a single emitter to generate fog.

The entire project was built using C++ and OpenGL on top of a foundational engine called Wolf, with all particle, emitter, rendering, and shader code developed exclusively by me.

►Game Jams - Itch.io

Unity Projects | Various Dates 2022-2024

I have developed several Unity projects in my spare time and for game jams hosted on itch.io, showcasing my passion for game design and my ability to solve problems effectively under tight deadlines. My itch.io profile features a variety of these smaller projects that I created as part of my ongoing learning process.

►Billy the Blueberry

Project Overview: Billy the Blueberry is a physics movement platform shooter where Billy combats small grubs and giant worms using a grappling hook that attaches to both enemies and environments. It features art all made by me, and was submitted for the GMTK game jam 2024.

During this project, I learned a lot about the pros and cons of physics-based movement, and had a lot of fun practicing making simple pixel art assets.

►Oubliette

Project Overview: Oubliette features 3D graphics all made by me. I wanted to create a turn-based card game with minimal UI and a focus on in-game world UI elements. In this game, the player is stationary as the environment and enemies move around them.

I learned a lot about in-world UI with this project and how much it relies on animation and feedback. I got to use Blender extensively to make the assets for this project.

►Beacon

Project Overview: Beacon is a simple text-based game that allows the player to explore a town in a choose-your-own-adventure style. It emulates the feel of an old computer terminal game and was my first game jam entry.

It is very buggy, but it was my first opportunity to apply my coding skills to a project I had imagined. This experience taught me a lot about feature creep as I attempted to implement quests, inventory, NPCs, and randomly generated locations—most of which had their fair share of bugs.

►Shader & Lighting Experiments

Project Overview: This was not a game jam entry but a series of experiments focusing on shaders, lighting, animations, rigging, and camera angles. I explored static camera angles inspired by classic Resident Evil games, created a water shader with foam effects, and experimented with various lighting types.

Through these experiments, I learned the intricacies of lighting systems, and the advantages of using baked lighting or directional lights over more resource-intensive options like spotlights or point lights.

►Shinjitsu no Megane

Project Overview: *Shinjitsu no Megane* is an experimental game where I explored alternative forms of animation and unique camera effects. By using pictures of myself and implementing layer masks, I was able to create alternative versions of environments, allowing for dynamic transitions between different realities. One of the key visual elements is an effect where the lenses of a pair of glasses reveal an alternate world as the player character dons them. This was achieved by carefully synchronizing the animation and mask transitions to enhance the player's sense of immersion and discovery.

►Robertson Library Perpetual Access Rights Tracker - GitHub

Desktop Application | October 2023 – April 2024

During my time working on the Robertson Library Perpetual Access Rights Tracker, my team and I developed a comprehensive technical document over the course of four months. This document included detailed requirements, roadmaps, diagrams, and thorough planning, which laid the groundwork for the application's creation during the winter semester.

My primary focus in this project was on implementing file upload and export functionality, along with developing multithreaded loading processes to enhance performance. I also contributed significantly to the UI design, ensuring a cohesive user experience by integrating the various components developed by other team members into a unified application.
